ゲームの目的

オンプレ環境から AWS への移行を進め、Well-Architected カード（勝利点）を集めます。 WAカードが全て無くなった瞬間にゲーム終了。勝利点の合計が最も多いプレイヤーが勝ちです。

ゲームの準備

各プレイヤーはオンプレカード10枚に加え、コンソール（コストフリービルダー）から 2枚を受け取り、よくシャッフルして山札にします。そこから5枚引いて手番を開始します。

3種類のカード

  • オンプレカード（×10）＋コンソールから2枚：初期デッキ（計12枚）。オンプレはクレジットを生みません。 デッキを薄くするためビルドフェーズで破棄していく対象です。 一部（Virtual Machine ⇔ Bare Metal Host、Corporate Identity Provider ⇔ AWS IAM Identity Center）は AWSカードとのシナジー効果を持ちます。
  • AWSカード（Builder）：クレジットはここから得ます（カード中央のオレンジ丸の数字）。 コストフリー（無料取得）とコストあり（クレジット消費）の2種。 場に出すとプライマリ効果（常時）、条件（同カテゴリ／特定カードが場にある等）を満たすと セカンダリ効果（シナジー）も発動します。
  • Well-Architected カード：勝利点（1pt=コスト3 / 3pt=コスト8）。1ptが残る間は3ptを買えません。

ターンの流れ

  1. ① ビルドフェーズ：
    ・手札のAWSカードが3枚以上ある場合、オンプレを1枚リタイア（このゲーム中は使えなくなる／除去）できます（効果でドローする前に）。
    ・手札のカードを場に出して +$ / +ドロー / +購入 を得る。
  2. ② 購入フェーズ（基本1購入・効果で増加）：
    (1) コストフリービルダーを取得 (2) コストありビルダーを購入 (3) WAを購入 (4) コンソール山札トップをブラインド取得（拒否不可）
  3. ③ エンドフェーズ：手札・場を全て捨て札へ → 5枚ドロー → 次のプレイヤーへ。

勝敗

勝利点の合計 → 同点なら AWSカード枚数が多い方 → それも同数なら引き分け。

攻略ヒント

  • 序盤〜中盤は compute / containers を中心に AWS カードを集め、オンプレを破棄してデッキを薄く。
  • シナジー（EC2複数、同カテゴリ、S3+CloudFront 等）を狙うとクレジットが伸びる。
  • クレジットが安定して8以上になったら3ptを一気に。
  • +購入エフェクトは1ターンに複数取得でき価値が高い。
